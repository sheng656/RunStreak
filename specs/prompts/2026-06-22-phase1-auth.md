# Prompt Log — Phase 1: Authentication & Database Setup

**Date:** 2026-06-22

## User Prompt
```
Phase 1 — implement the auth service (login/register/refresh/logout), EF Core initial migration, and provision the Azure SQL Free tier database.
```

## Description of Work
1. **DTO Creation:** Added `RegisterRequest`, `LoginRequest`, `UserDto`, and `AuthResponse` under `backend/RunStreak.Api/DTOs/Auth/` to transport and validate authentication requests using data annotations.
2. **Auth Service Implementation:**
   - Created `IAuthService` and `AuthService` under `backend/RunStreak.Api/Services/`.
   - Wired up C# standard `PasswordHasher<User>` for password PBKDF2 hashing.
   - Built access token generator (JwtSecurityToken) utilizing issuer, audience, key, and expiry configuration settings.
   - Implemented crypto-random refresh token generator storing SHA-256 hashes in SQL Server.
   - Implemented token rotation (rotate-on-use) and token reuse detection (family poisoning revocation of all active tokens).
3. **DI Registration:** Wired up services and password hasher in `Program.cs`.
4. **Auth Controller Implementation:**
   - Created `AuthController` under `backend/RunStreak.Api/Controllers/`.
   - Mapped endpoints `/api/auth/register`, `/api/auth/login`, `/api/auth/refresh`, and `/api/auth/logout`.
   - Set up cookie emission (HttpOnly for refresh tokens scoped to refresh endpoint path, non-HttpOnly for double-submit CSRF tokens).
   - Enforced double-submit CSRF verification header (`X-CSRF-Token` header matching `csrf_token` cookie) on token refresh.
   - Applied the preconfigured `login` rate-limiting policy on the login endpoint.
5. **Database Migration:** Installed global `dotnet-ef` tool and generated the initial migration `InitialCreate`.
6. **Unit Testing:** Created `AuthServiceTests.cs` in `RunStreak.Tests` with 9 passing tests verifying all auth logic using Entity Framework Core InMemory provider.
7. **Azure SQL DB Documentation:** Authored a provisioning guide `azure_sql_instructions.md` containing portal steps and Azure CLI commands.
